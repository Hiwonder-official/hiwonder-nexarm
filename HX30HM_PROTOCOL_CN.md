# HX-30HM 串行总线舵机通信协议

> 基于固件源码 `HX_30HM.c / HX_30HM.h / HX_30HM_Def.h` 整理
> 适用于 NexArm 六自由度机械臂

---

## 1. 物理层

| 参数 | 值 |
|------|----|
| 接口类型 | 单线半双工 UART |
| 波特率 | 1,000,000 bps (1 Mbps) |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | 无 |
| 逻辑电平 | TTL 3.3V |
| 总线拓扑 | 菊花链（多舵机共用一条信号线） |

发送完最后一个字节后，主机须等待至少 **20 µs** 再切换为接收模式。

---

## 2. 数据帧格式

### 2.1 指令帧（主机 → 舵机）

```
[0xFF] [0xFF] [ID] [LEN] [CMD] [PARAM...] [CHECKSUM]
```

| 字段 | 字节数 | 说明 |
|------|--------|------|
| 0xFF 0xFF | 2 | 帧头，固定值 |
| ID | 1 | 舵机 ID（0x01–0xFD），0xFE = 广播 |
| LEN | 1 | 后续字节数 = 参数数 + 2（CMD + CHECKSUM） |
| CMD | 1 | 指令编号 |
| PARAM | N | 指令参数（可为 0 字节） |
| CHECKSUM | 1 | 校验和 |

**校验和计算：**

```
CHECKSUM = (~(ID + LEN + CMD + PARAM[0] + ... + PARAM[N-1])) & 0xFF
```

### 2.2 应答帧（舵机 → 主机）

```
[0xFF] [0xFF] [ID] [LEN] [ERR] [PARAM...] [CHECKSUM]
```

| 字段 | 说明 |
|------|------|
| ERR | 错误标志位（见下表） |
| PARAM | 读取到的数据（仅 READ 指令有效） |

**错误标志位：**

| 位 | 含义 |
|----|------|
| bit0 | 输入电压错误 |
| bit1 | 角度超限 |
| bit2 | 过热 |
| bit3 | 指令范围错误 |
| bit4 | 校验和错误 |
| bit5 | 过载 |
| bit6 | 指令错误 |
| bit7 | 保留 |

## 3. 有符号数编码规则（符号-绝对值）

HX-30HM **不使用** 补码，采用自定义的符号-绝对值格式：

| 场景 | 规则 |
|------|------|
| 正数 | 直接使用原始值 |
| 负数 | `(-原值) OR (1 << 符号位)` |

不同寄存器的符号位位置：

| 寄存器 | 符号位 | 量程 |
|--------|--------|------|
| 位置 (Goal/Present Position) | bit15 | ±30719 |
| 速度 (Goal/Present Speed) | bit15 | ±3400 |
| 负载 (Present Load) | bit10 | ±1023 |
| PWM 速度 | bit10 | ±1000 |
| 位置偏移 | bit11 | ±2047 |

**编码/解码示例（C 语言）：**

```c
// 编码（写入舵机）
uint16_t encode_signed(int16_t val, uint8_t sign_bit) {
    return val < 0 ? (uint16_t)((-val) | (1U << sign_bit)) : (uint16_t)val;
}

// 解码（读出舵机）
int16_t decode_signed(uint16_t raw, uint8_t sign_bit) {
    return (raw & (1U << sign_bit))
        ? -(int16_t)(raw & ~(1U << sign_bit))
        : (int16_t)raw;
}
```

**示例：**

```
速度 = -500（bit15 符号位）
编码 = 500 | (1 << 15) = 0x81F4  →  L=0xF4, H=0x81

速度 = +500
编码 = 500 = 0x01F4  →  L=0xF4, H=0x01

读出原始值 0x81F4（bit15=1）→ 解码 = -(0x81F4 & 0x7FFF) = -500
```

## 4. 指令集

| 指令名 | CMD | 方向 | 有应答 | 说明 |
|--------|-----|------|--------|------|
| PING | 0x01 | 主→舵 | 是 | 检测舵机是否在线 |
| READ | 0x02 | 主→舵 | 是 | 读取寄存器 |
| WRITE | 0x03 | 主→舵 | 否* | 立即写入寄存器 |
| REG_WRITE | 0x04 | 主→舵 | 否* | 暂存写入（等 ACTION 执行） |

| SYNC_READ | 0x82 | 主→多舵 | 是×N | 同时读多个舵机相同寄存器 |

## 9. 各指令详解与数据包示例

### 9.1 PING — 检测舵机（ID=1）

```
发送: FF FF 01 02 01 FB
接收: FF FF 01 02 00 FC   (ERR=0x00 表示正常)
```

校验和：`~(0x01 + 0x02 + 0x01) & 0xFF = ~0x04 & 0xFF = 0xFB`

---

### 9.2 READ — 读取寄存器

格式：`[CMD=0x02] [起始地址] [读取字节数]`

#### 读当前位置（地址 56，2字节）

```
发送: FF FF 01 04 02 38 02 BF
                  |  └─ 读 2 字节
                  └──── 起始地址 56 (0x38) = 当前位置 L

接收: FF FF 01 04 00 PL PH CS
                     |  └─ 位置高字节
                     └──── 位置低字节
```

解码：`raw = PL | (PH << 8)`，再用符号-绝对值解码（符号位 bit15）

#### 读位置+速度（地址 56，4字节）

```
发送: FF FF 01 04 02 38 04 BD

接收: FF FF 01 06 00 PL PH SL SH CS
```

#### 读电压+温度（地址 62，2字节）

```
发送: FF FF 01 04 02 3E 02 B9

接收: FF FF 01 04 00 VL TL CS
                     |  └─ 温度 (°C)
                     └──── 电压 (× 0.1 V)
```

#### 读电流（地址 69，2字节）

```
发送: FF FF 01 04 02 45 02 B2

接收: FF FF 01 04 00 CL CH CS
```

#### 读负载（地址 60，2字节）

```
发送: FF FF 01 04 02 3C 02 BB

接收: FF FF 01 04 00 LL LH CS
```

解码：使用符号位 bit10。

#### 读运动状态（地址 66，1字节）

```
发送: FF FF 01 04 02 42 01 B6

接收: FF FF 01 03 00 MS CS
                     └─ 0=已停止，1=运动中
```

---

### 9.3 WRITE — 写入寄存器

格式：`[CMD=0x03] [起始地址] [数据...]`

#### 使能力矩（地址 40）

```
发送: FF FF 01 04 03 28 01 CF
                  |  └─ 值=1（使能）
                  └──── 地址 40 (0x28) = 力矩使能
```

#### 禁用力矩（自由拖动）

```
发送: FF FF 01 04 03 28 00 D0
```

#### 归零校准（当前物理位置作为零点）

```
发送: FF FF 01 04 03 28 80 50
                        └─ 128 = 归零指令
```

#### 写目标位置 2048（中间位置，约 150°）

```
2048 = 0x0800  →  L=0x00, H=0x08

发送: FF FF 01 05 03 2A 00 08 C3
                  └──────── 地址 42 (0x2A) = 目标位置 L
```

#### 同时写加速度+位置+速度（7字节，地址 41 起）

对应 `servo_write_pos_ex`：一次写入 ACC(1B)、目标位置(2B)、0x00 0x00、目标速度(2B)。

```
格式: [ACC][POS_L][POS_H][0x00][0x00][SPD_L][SPD_H]

示例: acc=100, pos=2048, speed=1000
  acc = 100 = 0x64
  pos = 2048 = 0x0800 → L=0x00, H=0x08
  speed = 1000 = 0x03E8 → L=0xE8, H=0x03

发送: FF FF 01 0A 03 29 64 00 08 00 00 E8 03 [CS]
                  └──── 地址 41 (0x29) = ACC
```

#### 设置工作模式

```
# 位置模式
发送: FF FF 01 04 03 21 00 D7

# 闭环电机模式
发送: FF FF 01 04 03 21 01 D6

# 开环电机模式
发送: FF FF 01 04 03 21 02 D5
```

#### 写 PWM 速度（电机模式，地址 44，符号位 bit10）

```
speed = 500  → encode = 0x01F4 → L=0xF4, H=0x01
发送: FF FF 01 05 03 2C F4 01 F8

speed = -500 → encode = 500 | (1<<10) = 0x05F4 → L=0xF4, H=0x05
发送: FF FF 01 05 03 2C F4 05 F4
```

#### 写最大力矩（地址 48）

```
torque = 800 = 0x0320 → L=0x20, H=0x03

发送: FF FF 01 05 03 30 20 03 A6
                  └──── 地址 48 (0x30) = 最大力矩 L
```

#### 写位置偏移（地址 31，符号位 bit11）

```
offset = -100 → encode = 100 | (1<<11) = 2148 = 0x0864 → L=0x64, H=0x08

发送: FF FF 01 05 03 1F 64 08 [CS]
                  └──── 地址 31 (0x1F) = 位置偏移 L
```

---

### 9.4 REG_WRITE + ACTION — 暂存写入后同步执行

先对每个舵机发送 REG_WRITE 暂存数据，再广播 ACTION 同时执行。

```
# 暂存舵机 1 的目标位置 pos=1000（acc=0, speed=0）
发送: FF FF 01 09 04 29 00 E8 03 00 00 00 00 [CS]

# 暂存舵机 2 的目标位置 pos=2048
发送: FF FF 02 09 04 29 00 00 08 00 00 00 00 [CS]

# 广播 ACTION，所有暂存写入同时执行
发送: FF FF FE 02 05 FA
```

---

### 9.5 SYNC_WRITE — 批量写多个舵机

`CMD=0x83`，参数格式：`[起始地址] [每舵机数据长度] [ID1] [数据...] [ID2] [数据...] ...`

**示例：同时设置 3 个舵机的 acc+pos+speed（7字节，地址 41 起）**

```
舵机1: id=1, acc=100, pos=1000, speed=500
舵机2: id=2, acc=100, pos=2048, speed=500
舵机3: id=3, acc=100, pos=3000, speed=500

pos1000 = 0x03E8 → L=E8, H=03
pos2048 = 0x0800 → L=00, H=08
pos3000 = 0x0BB8 → L=B8, H=0B
speed500 = 0x01F4 → L=F4, H=01

发送: FF FF FE [LEN] 83 29 07
      01  64  E8 03  00 00  F4 01
      02  64  00 08  00 00  F4 01
      03  64  B8 0B  00 00  F4 01
      [CS]

LEN = 2 + (1+7)×3 + 2 = 28 = 0x1C
```

---

### 9.6 SYNC_READ — 批量读多个舵机

`CMD=0x82`，参数：`[起始地址] [字节数] [ID1] [ID2] ...`

每个舵机独立回复，主机依次接收 N 个应答帧。

**示例：读取舵机 1、2、3 的当前位置（2字节）**

```
发送: FF FF FE 07 82 38 02 01 02 03 [CS]
                     |  |  └──── ID 列表
                     |  └─────── 读 2 字节
                     └────────── 起始地址 56 = 当前位置 L

接收（3个独立应答）:
FF FF 01 04 00 PL1 PH1 CS1
FF FF 02 04 00 PL2 PH2 CS2
FF FF 03 04 00 PL3 PH3 CS3
```

**示例：读取 6 个舵机的位置+速度+负载+电压+温度（8字节，地址 56 起）**

```
发送: FF FF FE 0A 82 38 08 01 02 03 04 05 06 [CS]

每个应答包含: [POS_L][POS_H][SPD_L][SPD_H][LOAD_L][LOAD_H][VOLTAGE][TEMPERATURE]
```

解码规则：
- 位置（bit15 符号位）：`decode_signed(POS_L | (POS_H<<8), 15)`
- 速度（bit15 符号位）：`decode_signed(SPD_L | (SPD_H<<8), 15)`
- 负载（bit10 符号位）：`decode_signed(LOAD_L | (LOAD_H<<8), 10)`
- 电压：raw × 0.1 V
- 温度：raw °C

---

### 9.7 RESET — 恢复出厂设置

```
发送: FF FF 01 02 06 F6
```

> **警告**：执行后 ID 恢复为 1，波特率恢复 1 Mbps，所有 EEPROM 参数恢复默认值。

## 11. 校验和计算示例

```
数据包: FF FF 01 04 02 38 02 [CS]
各字节之和 = 0x01 + 0x04 + 0x02 + 0x38 + 0x02 = 0x41
CS = (~0x41) & 0xFF = 0xBE
```

```
PING 包: FF FF 01 02 01 [CS]
各字节之和 = 0x01 + 0x02 + 0x01 = 0x04
CS = (~0x04) & 0xFF = 0xFB
```

## 13. NexArm ESP32 通信协议（上位机与 ESP32 之间）

ESP32 在 USB 串口上使用自己的帧格式与上位机通信，内部再转换为 HX-30HM 舵机总线指令。

### 帧格式

```
[0xFF] [0xFF] [0xFF] [LEN] [CMD] [ARGS...] [CHECKSUM]
```

注意：帧头为三个 0xFF（与舵机的两个 0xFF 不同）。

校验和 = `(~(0xFF + LEN + CMD + ARGS...)) & 0xFF`

### 指令表

| CMD | 参数 | 有应答 | 对应 HX-30HM 操作 | 说明 |
|-----|------|--------|-------------------|------|
| 56 | `[ACC, SPD_L, SPD_H]` | 否 | REG_WRITE 全部6舵机的 acc+speed | 设置运动加速度（0–254）和速度（0–3400） |
| 68 | `[0x01]` / `[0x00]` | 否 | — | 进入(1)/退出(0) LeRobot 桥接模式（仅从臂） |
| 96 | — | 是，12字节 | SYNC_READ 6舵机当前位置 | 读取全部6关节位置（6 × int16 小端序） |
| 97 | 12字节（6 × int16 小端序） | 否 | SYNC_WRITE 6舵机目标位置 | 同时写入全部6关节目标位置 |
| 98 | `[0x01]` / `[0x00]` | 否 | WRITE 力矩使能 × 6 | 使能(1)/禁用(0)全部6个舵机的力矩 |

### CMD 96/97 位置编码

位置值为 12 位原始值（0–4095），以 int16 小端序打包：

```python
import struct

# 编码 6 个位置（写入）
data = b"".join(struct.pack("<h", p) for p in positions)  # 12 字节

# 解码 6 个位置（读出）
positions = [struct.unpack_from("<h", reply, i*2)[0] for i in range(6)]
```
